# bill-record-api Design

## Goal

Build a personal bookkeeping and finance backend for one or more users. The API
must support account management, income/expense/transfer recording, budgets, and
monthly reports with JWT-secured access.

## Technical Constraints

- Java 21, Spring Boot 3.3, Maven.
- Package root: `com.hwiyn.billrecord`.
- Layers: `controller`, `service`, `repository`, `entity`, `dto`, `mapper`,
  `config`, `exception`, `security`.
- PostgreSQL 16 with Flyway migrations.
- Spring Data JPA for persistence.
- Spring Security with jjwt JWT access tokens.
- springdoc-openapi exposes Swagger UI and OpenAPI JSON.
- MapStruct maps entities to DTOs.
- Lombok reduces boilerplate for simple data classes.
- JUnit 5 for unit tests and Testcontainers PostgreSQL for integration tests.
- Dockerfile and Docker Compose are part of local/dev deployment.
- Money is always `BigDecimal`/PostgreSQL `numeric(19,2)`. `float` and
  `double` are not used for persisted or API money values.
- Default currency is `CNY`.
- Persisted timestamps are UTC `Instant`; API timestamps are ISO-8601 strings.

## Data Model

All primary keys are UUIDs generated by the application. All user-owned tables
include `user_id`, `created_at`, and `updated_at`.

### users

Stores API identities.

| Column | Type | Rules |
| --- | --- | --- |
| `id` | uuid | primary key |
| `email` | varchar(254) | required, lowercase, unique |
| `password_hash` | varchar(120) | BCrypt hash |
| `display_name` | varchar(80) | required |
| `default_currency` | char(3) | required, default `CNY` |
| `role` | varchar(30) | required, default `USER` |
| `enabled` | boolean | required, default `true` |
| `created_at` | timestamptz | required |
| `updated_at` | timestamptz | required |

### accounts

Stores cash, bank, credit card, and other balance containers.

| Column | Type | Rules |
| --- | --- | --- |
| `id` | uuid | primary key |
| `user_id` | uuid | required, references `users(id)` |
| `name` | varchar(80) | required, unique per user |
| `type` | varchar(30) | `CASH`, `BANK`, `CREDIT_CARD`, `INVESTMENT`, `OTHER` |
| `currency` | char(3) | required, default `CNY` |
| `opening_balance` | numeric(19,2) | required |
| `current_balance` | numeric(19,2) | required |
| `archived` | boolean | required, default `false` |
| `created_at` | timestamptz | required |
| `updated_at` | timestamptz | required |

### categories

Stores income and expense labels. System defaults are copied per user so users
can rename or archive categories without affecting others.

| Column | Type | Rules |
| --- | --- | --- |
| `id` | uuid | primary key |
| `user_id` | uuid | required, references `users(id)` |
| `name` | varchar(80) | required, unique per user/type |
| `type` | varchar(20) | `INCOME` or `EXPENSE` |
| `color` | varchar(20) | optional UI color token |
| `icon` | varchar(40) | optional UI icon token |
| `archived` | boolean | required, default `false` |
| `created_at` | timestamptz | required |
| `updated_at` | timestamptz | required |

### transactions

Stores ledger entries. A transfer is one logical transaction with both
`account_id` and `transfer_account_id`.

| Column | Type | Rules |
| --- | --- | --- |
| `id` | uuid | primary key |
| `user_id` | uuid | required, references `users(id)` |
| `account_id` | uuid | required, source account |
| `transfer_account_id` | uuid | required only for `TRANSFER` |
| `category_id` | uuid | required for `INCOME`/`EXPENSE` |
| `type` | varchar(20) | `INCOME`, `EXPENSE`, `TRANSFER` |
| `amount` | numeric(19,2) | required, positive |
| `currency` | char(3) | required, default `CNY` |
| `transaction_time` | timestamptz | required |
| `note` | varchar(500) | optional |
| `created_at` | timestamptz | required |
| `updated_at` | timestamptz | required |

Balance effects:

- `INCOME`: add `amount` to `account.current_balance`.
- `EXPENSE`: subtract `amount` from `account.current_balance`.
- `TRANSFER`: subtract from `account_id`, add to `transfer_account_id`.
- Updates and deletes reverse the old effect before applying the new effect.

### budgets

Stores monthly category budgets.

| Column | Type | Rules |
| --- | --- | --- |
| `id` | uuid | primary key |
| `user_id` | uuid | required, references `users(id)` |
| `category_id` | uuid | required, expense category |
| `month` | date | first day of month, UTC calendar |
| `amount` | numeric(19,2) | required |
| `currency` | char(3) | required, default `CNY` |
| `created_at` | timestamptz | required |
| `updated_at` | timestamptz | required |

Unique constraint: one budget per user/category/month.

## API Design

Base path: `/api/v1`.

Authentication uses `Authorization: Bearer <accessToken>`.

### Auth and Profile

| Method | Path | Description |
| --- | --- | --- |
| `POST` | `/auth/register` | Create user and return access token |
| `POST` | `/auth/login` | Authenticate and return access token |
| `GET` | `/users/me` | Return current user profile |

### Accounts

| Method | Path | Description |
| --- | --- | --- |
| `POST` | `/accounts` | Create account |
| `GET` | `/accounts` | List active accounts |
| `GET` | `/accounts/{id}` | Get account |
| `PATCH` | `/accounts/{id}` | Rename/archive/update metadata |
| `DELETE` | `/accounts/{id}` | Archive account |

### Categories

| Method | Path | Description |
| --- | --- | --- |
| `POST` | `/categories` | Create category |
| `GET` | `/categories?type=EXPENSE` | List categories |
| `PATCH` | `/categories/{id}` | Rename/archive/update metadata |
| `DELETE` | `/categories/{id}` | Archive category |

### Transactions

| Method | Path | Description |
| --- | --- | --- |
| `POST` | `/transactions` | Create income, expense, or transfer |
| `GET` | `/transactions` | Filter by date, type, account, category, page |
| `GET` | `/transactions/{id}` | Get transaction |
| `PATCH` | `/transactions/{id}` | Update transaction and rebalance accounts |
| `DELETE` | `/transactions/{id}` | Delete transaction and rebalance accounts |

### Budgets

| Method | Path | Description |
| --- | --- | --- |
| `POST` | `/budgets` | Create monthly budget |
| `GET` | `/budgets?month=2026-06` | List monthly budgets with spend |
| `PATCH` | `/budgets/{id}` | Update budget amount |
| `DELETE` | `/budgets/{id}` | Delete budget |

### Reports

| Method | Path | Description |
| --- | --- | --- |
| `GET` | `/reports/monthly-summary?month=2026-06` | Income, expense, net, and account balances |
| `GET` | `/reports/category-spending?from=...&to=...` | Expense totals grouped by category |
| `GET` | `/reports/cash-flow?from=...&to=...` | Daily income/expense/net series |

## Error Format

Errors use one JSON shape:

```json
{
  "timestamp": "2026-06-09T00:00:00Z",
  "status": 400,
  "code": "VALIDATION_FAILED",
  "message": "Request validation failed",
  "path": "/api/v1/accounts",
  "fieldErrors": [
    {"field": "name", "message": "must not be blank"}
  ]
}
```

   README baseline. Completed in commit `e192ce6`.
2. **Shared Infrastructure**: money value object, JWT service, UTC clock,
   OpenAPI security metadata. Completed in commit `4d141d0`.
3. **Auth and Users**: users migration/entity/repository, registration, login,
   `/users/me`, security filter, global errors, integration test.
4. **Accounts and Categories**: migrations/entities/repositories, CRUD APIs,
   default category seeding after registration.
5. **Transactions**: transaction CRUD, account balance updates, filters,
   transfer handling, integration tests.
6. **Budgets and Reports**: monthly budgets, spending progress, summary and
   cash-flow report queries.
7. **Documentation and Release Hardening**: bilingual README with complete HTTP
   examples, generated OpenAPI verification, Docker Compose app service, and
   CI-friendly test profile.
